# Frontend User Guide

## Мета
Цей документ описує, як швидко запустити frontend-частину, перевірити основні сценарії
та показати її на демонстрації або захисті.

## Варіанти запуску

### Локально однією командою
```powershell
.\run.ps1
```

Скрипт відкриває:
- backend на `http://127.0.0.1:8000`
- frontend на `http://127.0.0.1:5173`

### Локально вручну
1. Запустити backend:
```powershell
uvicorn src.main:app --reload
```
2. У другому терміналі запустити frontend:
```powershell
cd frontend
npm install
npm run dev
```

### Через Docker
```powershell
docker compose up --build
```

У контейнерному режимі frontend доступний на:
- `http://127.0.0.1:5173`

## Основні сторінки

### Home `/`
- показує загальний showcase-екран;
- виводить стан API;
- показує кількість категорій і публічних оголошень;
- відображає частину публічного каталогу.

### Catalog `/catalog`
- показує публічні схвалені оголошення;
- підтримує пошук за ключовими словами;
- підтримує фільтр за категорією;
- підтримує фільтр за мінімальною та максимальною ціною;
- підтримує сортування за датою або ціною;
- відображає галерею зображень для оголошень.

### Workspace `/workspace`
- містить логін і реєстрацію;
- показує дані поточного користувача;
- дозволяє створювати оголошення;
- дозволяє додавати URL зображень до оголошення;
- показує власні оголошення;
- показує повідомлення;
- для `admin` і `moderator` дає змогу створювати категорії.

## Рекомендований сценарій демонстрації
1. Відкрити `/` і показати, що frontend з'єднаний з API.
2. Перейти в `/catalog` і продемонструвати пошук, фільтри й сортування.
3. Перейти в `/workspace`.
4. Зареєструвати або залогінити користувача.
5. Створити оголошення з 1-2 `image_urls`.
6. Показати, що нове оголошення з'являється у власному списку.
7. За потреби схвалити оголошення через Swagger або акаунт модератора.
8. Повернутися в `/catalog` і показати оголошення вже в публічному каталозі.

## Що підготувати перед захистом
- переконатися, що backend працює без помилок;
- переконатися, що frontend відкривається на `http://127.0.0.1:5173`;
- мати хоча б кілька категорій і схвалених оголошень;
- мати 1 користувача для створення оголошень і 1 модератора для approval flow.

## Примітки
- dev frontend очікує API на `http://127.0.0.1:8000`;
- Docker frontend працює як production build через `nginx`;
- у production-контейнері API викликається через `/api/*` reverse proxy.
- Docker PostgreSQL публікується на `localhost:5433`, щоб не конфліктувати з локальним PostgreSQL.
