# Вимоги Проєкту

## Опис проблеми
Платформа для дошки оголошень потрібна для безпечного та структурованого розміщення товарів або послуг користувачами. У типових неформальних каналах продажу часто бракує категоризації, модерації, простого пошуку та контрольованої комунікації між продавцями й покупцями.

Система вирішує ці проблеми через керовану публікацію оголошень, модерацію контенту, ролі доступу, централізовані повідомлення та REST API для подальшої інтеграції з веб- або мобільним клієнтом.

## Цільова аудиторія
- Гість: переглядає схвалені оголошення та категорії.
- Користувач: реєструється, публікує оголошення, редагує їх, надсилає повідомлення.
- Модератор: перевіряє оголошення, схвалює або відхиляє їх.
- Адміністратор: керує категоріями, ролями та операційними правилами.

## Бізнес-вимоги
- Користувач може зареєструватися та увійти в систему.
- Користувач може створити, змінити й відправити оголошення на модерацію.
- Лише схвалені оголошення доступні для публічного перегляду.
- Повідомлення між користувачами дозволені лише для схвалених оголошень.
- Модератор має можливість відхилити оголошення з причиною.

## Нефункціональні вимоги
- Продуктивність: базові read-операції повинні відповідати менш ніж за 300 мс у локальному середовищі.
- Безпека: JWT-автентифікація, ролі доступу, валідація вхідних даних, захист від SQL injection через ORM.
- Надійність: покриття unit та integration тестами, healthcheck endpoint, ізоляція шарів.
- Зручність використання: OpenAPI документація, зрозумілі HTTP статуси, консистентні DTO.

## Обмеження
- Навчальний бюджет і стислі строки.
- Єдиний backend-сервіс без окремого брокера повідомлень.
- Базовий стек: Python, FastAPI, SQLAlchemy, PostgreSQL, Docker.

## Ключові use cases
1. Реєстрація користувача.
2. Авторизація користувача.
3. Створення оголошення і відправка на модерацію.
4. Перегляд черги модерації та прийняття рішення.
5. Публічний перегляд схвалених оголошень.
6. Надсилання повідомлення продавцю щодо оголошення.
7. Керування категоріями адміністратором.
