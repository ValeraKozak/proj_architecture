# Тестування

## Стратегія
- Unit тести покривають сервіси авторизації, категорій, оголошень, модерації та повідомлень.
- Integration тести перевіряють API-флоу через `TestClient`.

## Покриття сценаріїв
- успішна реєстрація та логін;
- заборона дублювання email;
- створення оголошення заблокованим користувачем;
- схвалення й відхилення оголошень;
- надсилання повідомлень тільки для схвалених оголошень;
- контроль доступу до модерації.

## Примітка
У наборі тестів використано параметризацію `pytest`, щоб компактно перевірити багато валідних і невалідних варіантів вхідних даних.
